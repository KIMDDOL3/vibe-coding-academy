로그인은 어떻게 작동할까요?

버튼 하나 누르면 뒤에서 이런 일들이 벌어져요!

👤 사용자가 이메일과 비밀번호 입력

로그인 화면에서 내 정보를 입력해요. 비밀번호는 보안을 위해 ●●●● 로 가려져 보여요.

📨 프론트엔드 → 백엔드로 전송

입력한 정보가 인터넷을 통해 서버(백엔드)로 전달돼요. 택배처럼 정보가 배송되는 거예요!

🔍 백엔드가 데이터베이스에서 확인

서버가 냉장고(데이터베이스)에서 같은 이메일의 계정을 찾아봐요. 비밀번호가 맞는지 확인해요.

🔑 맞으면 JWT 토큰 발급

비밀번호가 맞으면 특별한 디지털 열쇠(JWT)를 만들어 줘요. 이 열쇠는 7일 후에 사라져요.

💾 프론트엔드가 토큰 저장

받은 열쇠를 브라우저에 저장해요. 다음에 또 방문해도 다시 로그인 안 해도 돼요!

🎉 강의 페이지로 이동!

로그인 성공! 이제 학습 진도도 저장되고 내 계정으로 모든 기능을 쓸 수 있어요.

어떤 페이지들이 있나요?

총 7개의 페이지로 이루어진 웹앱이에요

/ (홈)

🏠 랜딩 페이지

처음 방문했을 때 보이는 화면이에요. "바이브 코딩이 뭔지" 소개하고 수강 시작 버튼이 있어요.

/roadmap

🗺️ 로드맵 페이지

전체 학습 여정을 지도처럼 보여줘요. 어떤 순서로 배워야 하는지 한눈에 볼 수 있어요.

/courses

📚 강의 목록

3개 모듈의 모든 강의를 카드 형태로 보여줘요. 내 학습 진도율도 표시돼요.

/courses/:id

📖 강의 상세

선택한 모듈의 레슨 목록을 보여줘요. "수강 시작" 버튼으로 바로 시작할 수 있어요.

/courses/:id/lessons/:id

💻 레슨 페이지

실제 강의 내용을 읽고 코드 실습도 할 수 있어요. 완료 버튼을 누르면 다음 레슨으로!

/login

🔐 로그인

이미 계정이 있는 사람이 로그인하는 화면이에요. 이메일과 비밀번호를 입력해요.

/register

✏️ 회원가입

새 계정을 만드는 화면이에요. 이름, 이메일, 비밀번호를 입력하면 바로 시작할 수 있어요!

데이터는 어떻게 저장되나요?

냉장고(DB)에는 딱 두 종류의 정보만 저장해요

👤 users (회원 정보)
id 🔑 기본키 숫자
email 이메일 주소
password 암호화된 비밀번호
name 이름
created_at 가입 날짜
📊 progress (학습 진도)
id 🔑 기본키 숫자
user_id 👤 회원 번호
lesson_id 📖 레슨 번호
completed 완료 여부 (1/0)
updated_at 완료 날짜

📒 쉽게 말하면: users 테이블은 회원 명단 공책이고, progress 테이블은 출석부예요. "홍길동(user_id=1)이 레슨3(lesson_id=m1-l3)을 완료했다"는 식으로 기록해요!

API가 뭔가요?

프론트엔드와 백엔드가 대화하는 규칙이에요

POST /api/auth/register 회원가입

📮 이름·이메일·비밀번호를 보내면 → 새 계정을 만들고 로그인 열쇠(토큰)를 돌려줘요

POST /api/auth/login 로그인

🔐 이메일·비밀번호를 보내면 → 맞으면 열쇠(토큰)를 돌려줘요. 틀리면 오류!

GET /api/progress 진도 조회

📊 내 학습 열쇠를 보내면 → 내가 완료한 레슨 목록을 돌려줘요

POST /api/progress/:레슨ID 레슨 완료

✅ 레슨 번호를 보내면 → 출석부에 "완료!"라고 체크해요
